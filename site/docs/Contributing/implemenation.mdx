# Contributing to implementation

<LargeParagraph>
    Ready made components are what make HDS tick. You can contribute to HDS React and Core libraries by proposing new implementations and bugfixes to existing components.
</LargeParagraph>


<p>When contributing for implementation, create a <Link href="https://help.github.com/en/github/collaborating-with-issues-and-pull-requests/creating-a-pull-request" external>pull request</Link> to the <Link href="https://github.com/City-of-Helsinki/helsinki-design-system" external>HDS GitHub repository</Link>.</p>

### Guidelines for implementation

<li>HDS follows dev.hel.fi frontend coding standards: <Link href="https://dev.hel.fi/coding-standards#frontend-code-standards" external>Frontend coding standards</Link></li>

#### General guides

<li>Do not merge the pull request after it is accepted. The HDS team will merge it. Merging the PR has a side-effect which updates documentation site.</li>
<li>Remember to apply <Link href="https://github.com/City-of-Helsinki/helsinki-design-system/tree/master/packages/design-tokens/tokens">HDS Design Tokens</Link> when creating a component. Instead of specifying pixel values, colours, fonts or breakpoints by yourself, apply those from design tokens instead</li>
<li>Take a look at <Link href="https://dev.hel.fi/" external>Develop with Helsinki site</Link> for practical details for developing open source code for the City of Helsinki.</li>

#### HDS Technologies

<li>HDS React and HDS Core both apply the <Link href="https://storybook.js.org/">Storybook library</Link></li>
<li>HDS React is build with Typescript and SASS</li>
<li>HDS Core is build with pure JS and CSS</li>
<li>HDS applies yarn as its dependency manager</li>
<li>HDS applies git for version control with <Link href="https://git-lfs.github.com/">GIT-LFS support</Link></li>

#### Creating a new component

<li>Apply the Scaffolding yarn script when creating a new component: <Link href="https://github.com/City-of-Helsinki/helsinki-design-system/blob/master/packages/react/package.json#L17">Scaffolding</Link></li>
<li>For simple components, create both versions, HDS Core and HDS React.
  Create common styles to HDS core that you can then import to HDS React like in this <Link href="https://github.com/City-of-Helsinki/helsinki-design-system/blob/master/packages/react/src/components/textInput/TextInput.module.css">example</Link>.
  HDS React has HDS Core as its dependency.
</li>
<li>Add ready components to <Link href="https://github.com/City-of-Helsinki/helsinki-design-system/blob/master/packages/react/rollup.config.js">rollup.config.js</Link></li>

#### Testing

<li>Write unit tests for each component. Test rendering, accessibility and possible functionality of the component. HDS applies <Link href="https://jestjs.io/">Jest</Link> for unit testing </li>
<li>HDS applies <Link href="https://loki.js.org/">Loki</Link> for visual regression testing. Loki is integrated with storybook. Hence it's important to add a story for each different UI state of the component</li>
<li>It is a good idea to test the component independently in a fresh <Link href="https://github.com/City-of-Helsinki/hds-cra">Create React App</Link></li>
