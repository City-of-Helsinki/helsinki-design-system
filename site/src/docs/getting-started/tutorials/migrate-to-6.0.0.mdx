# Migrate to version 6.0.0

This guide helps you migrate from HDS v5 to v6. The v6 release focuses on tooling and platform upgrades. The main consumer-facing breaking changes are the removal of the **ErrorSummary** component, the requirement for **React 19**, and the bump of the supported **Node.js** version to 24.

<Notification label=";tldr; Check at least these" className="siteNotification">
  When updating to HDS 6.0.0 check and test at least these: ErrorSummary component is removed from both React and Core — use `<Notification type="error">` with the new `autofocus` prop instead; Node.js 24 is now the supported runtime (`engines.node` is `^24`); React 19 or later is required (`react` and `react-dom` peer dependencies are `^19.0.0`); the Figma Design Kit only fixes font rendering in components and is otherwise equivalent to v5.
</Notification>

## Breaking changes

### ErrorSummary component removed

The standalone **ErrorSummary** component has been removed from both the React and Core packages. The same pattern is now expressed with the **Notification** component using `type="error"`, which has been extended in 5.2.0 to support the error-summary use case directly:

- A new `autofocus` prop programmatically focuses the notification label after mount — useful after a failed form submission.
- `ul`, `li` and `a` styles are scoped to error-type notifications, so anchor-linked error lists render correctly without extra CSS.

#### React

```diff
- import { ErrorSummary } from 'hds-react';
+ import { Notification } from 'hds-react';
```

**Before / after example:**

```jsx
// ❌ v5 — ErrorSummary
<ErrorSummary label="Form contains the following errors">
  <ul>
    <li><a href="#first-name">Please enter your first name</a></li>
    <li><a href="#email">Please enter a valid email address</a></li>
  </ul>
</ErrorSummary>
```

```jsx
// ✅ v6 — Notification as error summary
<Notification
  type="error"
  label="Form contains the following errors"
  notificationAriaLabel="Error summary"
  autofocus
>
  <ul>
    <li><a href="#first-name">Please enter your first name</a></li>
    <li><a href="#email">Please enter a valid email address</a></li>
  </ul>
</Notification>
```

**Migration:** Replace each `<ErrorSummary>` usage with `<Notification type="error">`. Pass the same `label` prop, add `notificationAriaLabel` for the accessible region label, and set `autofocus` if you want focus to move to the summary after submit. To force focus on every submit attempt (e.g. when the same errors appear again), give the Notification a `key` derived from the submit counter so React re-mounts it.

#### Core

The `hds-error-summary` block and accompanying styles have been removed. Use the Notification block with the error modifier instead. Wrap your error list in the notification body — the new `ul / li / a` styles apply automatically.

### Node.js 24 required

HDS packages now declare `"engines": { "node": "^24" }`. The repository's `.nvmrc` pins the version used for local development and GitHub Actions.

**Migration:**
- Upgrade your build and CI environments to Node.js 24.
- If you use `nvm`, run `nvm use` in the repo root (or update your own `.nvmrc` to 24).
- Lower Node versions are no longer tested and may produce build or runtime warnings.

---

### React 19 required, Storybook 8 and Gatsby 5

HDS 6 requires **React 19** or later. The published `hds-react` package declares:

```json
"peerDependencies": {
  "react": "^19.0.0",
  "react-dom": "^19.0.0"
}
```

Package managers warn at install time if your application does not satisfy this range.

**Migration:** Upgrade `react` and `react-dom` to version 19 before or when upgrading to HDS 6. Follow the official [React 19 upgrade guide](https://react.dev/blog/2024/04/25/react-19-upgrade-guide) for application-level changes (removed legacy APIs, ref-as-prop changes, etc.).

HDS itself is also developed against **Storybook 8** and **Gatsby 5**. If you maintain a Storybook integration that loads HDS stories, or you fork the documentation site, update Storybook and Gatsby alongside HDS.

## Figma Design Kit

This guide will help you understand how to transition an existing project from using an older version of HDS Figma Design Kit to using the latest 6.0.0 version.

### What's changing?

HDS 6.0.0 is mainly a codebase React update. In Figma, the release is intended only to fix font rendering issues within components — otherwise it equals the v5 library set.

**Note:** Card, Footer and Table assets remain unreleased in this library, meaning they still show the same structure and style as in early v4, and they don't show yet the font updates introduced in v5. However, you can see the upcoming changes on them within the file documentation. Visit the file's Changelog page for more details.

### What to consider before updating to HDS Kit v6?

Update or swap libraries safely by first duplicating your design files.

### Fixed in Figma

- Forced new font rendering in all components using the library typography styles.

### How to safely update from v5 to v6?

#### 1. Duplicate your current design file

In order to avoid disruptions to your workflow, it is recommended that you duplicate your project files before accepting this update. This allows you to migrate to the latest HDS Design Kit while maintaining a reference to your original file. A guide for duplicating files can be found in Figma Learn.

#### 2. Review and accept updates

In your duplicate file, review and accept the HDS Design Kit library updates.

#### 3. Check results

A successful update should make the new font render correctly in all components that are using it. The update should not imply any structural changes or breakage. If you find any, please contact the HDS team.

### How can I continue using HDS v5?

Although we recommend using the latest HDS version, in case your project is not ready to take the new v6, then you can either:

a. Leave your files with pending update (no further tasks required)

b. Swap to the new separate HDS Design v5.2.0 fallback library (follow steps below)

In case you choose option b. your file will no longer receive v6 updates.

#### Swap library

1. Duplicate first your current design file.
2. Instead of taking the v6.0.0 updates, swap in assets/libraries the HDS Design Kit v6.0.0 library into HDS Design Kit v5.2.0. Swapping libraries allows you to quickly update instances in a file to use components from another library. The guide for swapping components can be found in Figma Learn.

### How to migrate from HDS v4 to v6?

If you are using the separate HDS v4 file in your designs, swap the v4 library to the new HDS Kit v6.0.0.

### Can I continue using HDS v4?

Due to the nature of this update (mainly React version update in codebase) we will still provide support for v4.
