# Migrate to version 6.0.0

This guide helps you migrate from HDS v5 to v6. The v6 release focuses on tooling and platform upgrades. The main consumer-facing breaking changes are the removal of the **ErrorSummary** component and the bump of the supported **Node.js** version to 24.

<Notification label=";tldr; Check at least these" className="siteNotification">
  When updating to HDS 6.0.0 check and test at least these: ErrorSummary component is removed from both React and Core — use `<Notification type="error">` with the new `autofocus` prop instead; Node.js 24 is now the supported runtime (`engines.node` is `^24`); HDS is now built and tested against React 19 (peerDependency range is unchanged: React 16.8+ through 19 still supported).
</Notification>

## Breaking changes

### ErrorSummary component removed

The standalone **ErrorSummary** component has been removed from both the React and Core packages. The same pattern is now expressed with the **Notification** component using `type="error"`, which has been extended in 5.2.0 to support the error-summary use case directly:

- A new `autofocus` prop programmatically focuses the notification label after mount — useful after a failed form submission.
- `ul`, `li` and `a` styles are scoped to error-type notifications, so anchor-linked error lists render correctly without extra CSS.

#### React

```diff
- import { ErrorSummary } from 'hds-react';
+ import { Notification } from 'hds-react';
```

**Before / after example:**

```jsx
// ❌ v5 — ErrorSummary
<ErrorSummary label="Form contains the following errors">
  <ul>
    <li><a href="#first-name">Please enter your first name</a></li>
    <li><a href="#email">Please enter a valid email address</a></li>
  </ul>
</ErrorSummary>
```

```jsx
// ✅ v6 — Notification as error summary
<Notification
  type="error"
  label="Form contains the following errors"
  notificationAriaLabel="Error summary"
  autofocus
>
  <ul>
    <li><a href="#first-name">Please enter your first name</a></li>
    <li><a href="#email">Please enter a valid email address</a></li>
  </ul>
</Notification>
```

**Migration:** Replace each `<ErrorSummary>` usage with `<Notification type="error">`. Pass the same `label` prop, add `notificationAriaLabel` for the accessible region label, and set `autofocus` if you want focus to move to the summary after submit. To force focus on every submit attempt (e.g. when the same errors appear again), give the Notification a `key` derived from the submit counter so React re-mounts it.

#### Core

The `hds-error-summary` block and accompanying styles have been removed. Use the Notification block with the error modifier instead. Wrap your error list in the notification body — the new `ul / li / a` styles apply automatically.

### Node.js 24 required

HDS packages now declare `"engines": { "node": "^24" }`. The repository's `.nvmrc` pins the version used for local development and GitHub Actions.

**Migration:**
- Upgrade your build and CI environments to Node.js 24.
- If you use `nvm`, run `nvm use` in the repo root (or update your own `.nvmrc` to 24).
- Lower Node versions are no longer tested and may produce build or runtime warnings.

---

### Internal upgrade: React 19, Storybook 8, Gatsby 5

HDS itself is now developed against **React 19**, **Storybook 8** and **Gatsby 5**. For consumers of the published packages this is mostly transparent — the `react` and `react-dom` peer dependency range is unchanged:

```json
"peerDependencies": {
  "react": "^16.8.0 || ^17 || ^18 || ^19",
  "react-dom": "^16.8.0 || ^17 || ^18 || ^19"
}
```

You can keep your application on React 17 or 18. That said, we recommend upgrading to React 19 to stay aligned with the version HDS is now tested against. If you maintain a Storybook integration that loads HDS stories, or you fork the documentation site, update Storybook and Gatsby alongside HDS.

**Migration:** No code changes are required if you stay on your current React major. If you upgrade React, follow the official [React 19 upgrade guide](https://react.dev/blog/2024/04/25/react-19-upgrade-guide) for any application-level changes (removed legacy APIs, ref-as-prop changes, etc.) — those are React's own breaking changes, not HDS's.

## Summary

| Area              | Change |
|-------------------|--------|
| **ErrorSummary**  | Removed from React and Core. Use `<Notification type="error">` with the new `autofocus` prop. |
| **Node.js**       | `engines.node` is now `^24`. Upgrade your build and CI environments. |
| **React / Storybook / Gatsby** | HDS is now built against React 19, Storybook 8 and Gatsby 5. The React peerDependency range is unchanged. |
