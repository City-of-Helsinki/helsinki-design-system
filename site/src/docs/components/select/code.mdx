### Passing options

Select accepts options as strings, objects, or HTML. The simplest format is an array of strings:

```jsx
<Select options={['Option 1', 'Option 2']} />
```

Sometimes the value shown to the user should be different than the stored value. Then the options should be objects:

```jsx
<Select options={[{ label: 'The label is shown to the user', value: 'The actual value' }]} />
```

Initially selected options can be set with the `value` property of the Select component or by setting the "selected" property of an option to `true`.

```jsx
// these props result in the same output
<Select options={["Option 1", "Option 2" ]} value="Option 2"/>
// as these props
<Select options={[{ label: "Option 1", selected:false }, { label: "Option 2", selected:true } ]} />
```

Options can also be individually hidden with the `visible` property or made unselectable with the `disabled` property.

#### Properties of the option object

| Property                     | Description                                                                | Values    | Default value |
| ---------------------------- | -------------------------------------------------------------------------- | --------- | ------------- |
| `disabled`                   | Can the option be selected or unselected.                                  | `boolean` | false         |
| `isGropLabel`                | Internal. Indicates is a group label option. Should not be edited.         | `boolean` | false         |
| `label`                      | Label, visible to the user.                                                | `string`  | -             |
| `selected`                   | Is the option selected.                                                    | `boolean` | false         |
| `value`                      | Value, not visible to the user.                                            | `string`  | false         |
| `visible`                    | Is the option visible. Options are hidden, for example, when filtered out. | `boolean` | false         |
| [Table 1: Option properties] |

Internally, all options are always converted to objects. Whenever options are passed to callbacks like `onChange` or `onSearch`, the options are in object format.

### Options as groups

Options can be grouped by providing a `groups` property. Groups have only `label` and `options` properties.

```jsx
<Select
  groups={[
    {
      label: 'Group 1 label',
      options: [{ label: 'Option 1' }, { label: 'Option 2' }],
    },
    {
      label: 'Group 2 label',
      options: [{ label: 'Option 1' }, { label: 'Option 2' }],
    },
  ]}
/>
```

If there is only one group and the label is empty, the label is not shown. Multiple groups should all have labels. If the Select has `multiSelect` set to `true`, the group label can also be selected. Selecting it selects all options in that group. Selecting all options individually also selects the group's label.

Internally, all options are set into a group even if the `groups` property is not passed in props. The first option of all groups is always an option with `option.isGroupLabel` set to `true`. Even if the label was not given.

This ensures the data is always in the same format internally and in all callbacks, despite what format options were passed to the component.

The data and selected options are easily handled with <InternalLink href="#utility-functions">utility functions</InternalLink>.

### Options as HTML

Options can also be defined the same way the native `<select>` element accepts them.

```jsx
// some properties are omitted to make the example clearer
<Select>
  <optgroup label="Group 1">

    <option value="opt2">Option 2</option>
  </optgroup>
  <optgroup label="Group 2">
    <option value="opt3">Option 3</option>
    <option value="opt4">Option 4</option>
  </optgroup>
</Select>
```

### Changing options at runtime

Options can be changed by passing a new set of `groups` or `options` as props to the component. This will naturally create a new instance of the component and clear the component's internal state.

The `onChange` callback can also return a new set of options. This is the easiest way to manipulate existing options. It can be used when there should be a <ExternalLink href="https://hds.hel.fi/storybook/react/?path=/story/components-select--with-min-max">minimum or maximum number of options</ExternalLink> or if at least one option must be selected:

```jsx
import { Select, iterateAndCopyGroup } from 'hds-react';

// some properties are omitted to make the example clearer
<Select
  invalid
  onChange={(selectedOptions, clickedOption, data) => {
    // undo invalid state after a selection has been made
    if (selectedOptions.length > 0) {
      return {
        invalid: false,
      };
    }
    // return a new set of options where the unselected is still selected
    return {
      groups: iterateAndCopyGroup(data.groups, (option) => {
        if (option.value === clickedOption.value) {
          return { ...option, selected: true };
        }
        return option;
      }),
    };
  }}
/>;
```

### Handling changes

When the user changes the selected options, the `onChange` is called. Other user actions do not trigger the callback. Not even searching or filtering. When using multiselect, it's advised to use the `onClose` callback to get the selections after the user has finished selecting options but any.

The callbacks has the following signature:

```jsx
// some properties are omitted to make the example clearer

// single select
<Select onChange={(selectedOptions, clickedOption, data) => {...} />;

// multiselect, use onClose to get the selections after the user closed the dropdown
<Select multiSelect onClose={(selectedOptions, clickedOption, data) => {...} />;
```

| Argument                                       | Description                                                                                                                                  | Values                                                                             | Default value |
| ---------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------- |
| `selectedOptions`                              | An array of selected options in object format.                                                                                               | `Option[]`                                                                         | []            |
| `clickedOption`                                | The option that was clicked. Can be a group label too. The option data is prior to the selection, so its `selected` property is not updated. | `Option` or `undefined` if `onClose`                                                | -             |
| `data`                                         | All data of the Select component.                                                                                                            | <InternalLink href="#the-data-property-in-the-callbacks">SelectData</InternalLink> | -             |
| [Table 2: Arguments of the onChange and onClose callbacks ] |

The `onChange` and `onClose` can return data that is stored immediately to the Select component. Returning any data is optional.

| Property                                                            | Description                                                                                                                | Values     | Default value |
| ------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | ---------- | ------------- |
| `groups`                                                            | Array of `groups` in the same format as passed to the Select component. Cannot be used with `options`.                     | `Option[]` | -             |
| `invalid`                                                           | The `invalid` property.                                                                                                    | `boolean`  | -             |
| `options`                                                           | Array of `options` in the same format as passed to the Select component. Cannot be used with `groups`.                     | `Option[]` | -             |

### Texts

The component has default texts in Finnish, English, and Swedish. The used language is passed in `texts.language`. The value can be `fi`, `en` or `sv`. Default is `fi`.

If additional texts are passed, those values are checked first and any missing texts are picked from the default texts with the given language.

Usually `label` and `placeholder` are given to instruct the user what to do.

```jsx
// some properties are omitted to make the example clearer
<Select
  texts={{
    language: 'en',
    label: 'Select something',
    placeholder: 'Choose one',
    language: 'en',
  }}
/>
```

Note that passed texts do not have language versions. If language changes, pass new texts and updated `language` property.

#### Text templates and interpolation

In some cases, the outputted text depends on the option that was selected or the number of selections. For example, when a search is finished, a notification is rendered for the screen reader: "Found 5 options for search term 'Banana'."

#### Updating texts

Texts can be changed with props, using the text function, and also at runtime when the `onChange` is called. If the `onChange` returns a `texts` property, the texts will be appended to the current texts.

This is a convenient way to show error or assistive texts after the user makes changes.

### Filtering

Options can be filtered with a function. If the `filter` property is set, the component will show an input field where the user can write text to filter options with.

HDS provides a `defaultFilter` that checks if the label of an option starts with given text. The filter function should return `true` or `false`. The returned value is set to the option's `visible` property, and options are then hidden. Group labels are not filtered. They are visible if any of their options are visible.

The `filter` function has the following signature:

```jsx
// some properties are omitted to make the example clearer
<Select filter={(option, filterValue) => {return true} />;
```

It is called with the following arguments:

| Argument                                     | Description                 | Values   |
| -------------------------------------------- | --------------------------- | -------- |
| `filterValue`                                | The text user has inputted. | `string` |
| `option`                                     | Current option to filter.   | `Option` |
| [Table 6: Arguments of the filter function ] |

### Searching

The user may also search for options. Select with a search should not have initial options before the user has searched for something. Unless the options are already selected. Selected options are always preserved, even if a new search result does not include them.
The `search` function is asynchronous and should return `groups`, `options` or an error.

The `search` function has the following signature:

```jsx
// some properties are omitted to make the example clearer
<Select search={(searchValue, selectedOptions, data)=>{...}} />;
```

It is called with the following arguments:

| Argument                                     | Description                       | Values                                                                             |
| -------------------------------------------- | --------------------------------- | ---------------------------------------------------------------------------------- |
| `searchValue`                                | The text user has inputted.       | `string`                                                                           |
| `selectedOptions`                            | Currently selected options.       | `Option[]`                                                                         |
| `data`                                       | All data of the Select component. | <InternalLink href="#the-data-property-in-the-callbacks">SelectData</InternalLink> |
| [Table 7: Arguments of the filter function ] |

The function should return either `groups` or `options`.

### Getting selections to parent state

The selected options can be set to parent state from within the `onChange` and/or `onClose` callbacks. The parent component should have a state for the selected options and a callback function to update the state.

The example below also displays a really simple validation. The user must select one option and if no correct selection is done, an error message is shown.

<Playground scope={{ Select }}>

```jsx
import { Select } from 'hds-react';

() => {
  const [onCloseSelections, setOnCloseSelections] = React.useState([]);

  const onClose = React.useCallback((selectedOptions, lastClickedOption, data) => {
    setOnCloseSelections(selectedOptions);
    return { invalid: selectedOptions.length !== 1, texts: { error: 'Choose only one' } }
  }, []);

  const [props] = React.useState({
    multiSelect: true,
    id: 'hds-select-component',
    clearable: true,
    required: true,
    options: [
      'First',
      'Second',
      'Third',
    ],
    texts: {
      label: 'Select one',
      language: 'en',
    },
    noTags: true,
  });
  
    return (
    <div style={{ display: 'flex', flexDirection: 'column', gap: '10px', maxWidth: '420px' }}>
      <label id="onclose-label" htmlFor="onclose">
        Selected values in parent state
      </label>
      <input
        placeholder="No selected values"
        type="text"
        id="onclose"
        aria-labelledby="onclose-label"
        readOnly
        value={onCloseSelections.map((option) => option.label).join(', ')}
      />
      <Select {...props} onClose={onClose} />
    </div>
  );
}
```

### useSelectStorage hook

The properties passed to the Select component must be memoized or React will render a new instance of the component when props change. This means user's selections could be lost when the component re-renders.

Properties and selected options may be stored to the parent component's state, and the user's changes have to be syncronized manually when the `onChange` is called.

The `useSelectStorage` is an utility hook that manages the memoisation and data updates automatically. It accepts the same properties as the Select function, except the "value" property. The selected values should not be reset again manually after initialisation.

```jsx
const storage = useSelectStorage({
  groups: [
    {
      label: 'Group 1',
      options: [...],
    },
    {
      label: 'Group 2',
      options: [...],
    },
  ],
  onChange: (selectedOptions) => {
    // handle changes as soon as they happen
  },
  onClose: (selectedOptions) => {
    // handle changes after closing the dropdown
  },
  multiSelect: true,
  filter: defaultFilter,
  disabled: false,
  open: false,
  invalid: false,
  id: 'hds-select-component',
});
<Select {...storage.getProps()} />;
```

### The data property in the callbacks

Callbacks receive all current data of the Select component. This includes the following properties of the component:

- `disabled`
- `filterFunction`
- `groups` (is an array of `{ options: Option[] }`)
- `invalid`
- `onBlur`
- `onChange`
